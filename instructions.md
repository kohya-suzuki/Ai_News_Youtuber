# Claude Code への開発指示書: スプレッドシートデータソース連携の実装

## 1. 目的

本プロジェクトは、Google スプレッドシートからニュース原稿や各種メタデータを取得し、音声合成（gTTS）および動画レンダリング（MoviePy v2.0）を経て、YouTube へのアップロードと Discord 通知までを完全自動化する「AIニュースYouTuberシステム」の開発です。

現在のシステムは、`test-main.py` にて擬似データ（mock_data）を用いたスタンドアロン動作（動画生成・YouTubeアップ、Discord通知）の検証が完了しています（MoviePyの座標判定に伴う既知の内部エラーはありますが、動画生成自体は正常に完了します）。

今回のタスクでは、`test-main.py` をベースに、**Google Sheets API を用いた本番用データソース連携ロジックを組み込んだ `main.py` を新規作成（またはリファクタリング）すること**を目的とします。

---

## 2. 現在の環境と前提条件

- **ランタイム:** Python 3.11+ (仮想環境 `./venv/bin/python`)
- **主要ライブラリ:** `moviepy` (v2.0.0.dev2 移行のバージョン), `gtts`, `google-api-python-client`, `google-auth-oauthlib`, `requests`
- **認証アセット:** - プロジェクトルートに `client_secrets.json` が配置済み。
  - OAuth 2.0 認証を通過すると `token.json` が生成される（または更新される）仕組みが実装済み。
- **ベースコード:** `test-main.py`（旧メインスクリプト。モックデータで動作するコードが格納されている）

## 3. データソース（Googleスプレッドシート）の仕様

### シート構成

- **対象ワークシート名:** `Main_Scripts`
- **データの単位:** **「1行 ＝ 1本の動画（番組）」**

### 列定義（A列 〜 K列）

各行は以下の11列で構成されています。プログラム内でのインデックス処理（`row[0]`〜`row[10]`）に正確にマッピングしてください。

1. **A列 (row[0]): 取得日時** -> データインサート時のタイムスタンプ（処理対象の判定用キー）
2. **B列 (row[1]): 曜日** -> 番組内アナウンスやタイトル・説明欄の動的挿入用
3. **C列 (row[2]): AIモデル名** -> 動画内「Powered by [モデル名]」の表示用
4. **D列 (row[3]): 番組全台本** -> TTS音声合成用のナレーションテキスト（**※複数ニュース分が格納**）
5. **E列 (row[4]): 要約リスト** -> YouTubeの動画説明欄（Description）に出力するトピックス概要
6. **F列 (row[5]): 見出し** -> 動画内左側の黄色いボードに動的表示するニュースタイトル（**※複数ニュース分が格納**）
7. **G列 (row[6]): 画像KWリスト** -> 中央ワイプ表示に使う画像を特定・検索するためのキーワード/パスのリスト
8. **H列 (row[7]): YouTube URL** -> **【重要】** アップロード成功時にシステムがURLを書き戻す列。空の行を「未処理」として扱い、二重実行を防止する。
9. **I列 (row[8]): ニュースタイトル** -> YouTubeの動画タイトルおよびDiscord通知のベース
10. **J列 (row[9]): ニュース取得元URL** -> 出典としてYouTube説明欄に記載するURL
11. **K列 (row[10]): 愚痴内容** -> 動画ラストの「中の人の本音」セグメント（音声およびテロップ）用のテキスト

## 4. Claude Code への具体的な実装要求

### タスク 1: 処理対象行の特定と読み込み

1. `googleapiclient.discovery.build("sheets", "v4", ...)` を使用し、`Main_Scripts` シートからデータを一括取得する。
2. 取得した行の中から、**「H列（YouTube URL）が空」かつ「データ（A列など）が存在する」最新の1行**、または**トリガーされた条件に合致する対象行**を1行だけ特定する。
3. すべての行が処理済み（すべてのH列にURLがある）の場合は、処理を安全にスキップ（早期リターン）するログを吐くこと。

### タスク 2: 複数ニュース要素のパース（構造化）

D列（全台本）、F列（見出し）、G列（画像KWリスト）には、1本の動画内で順番に紹介する**3本分（複数）のニュースデータ**が格納されています。

1. 各セル内のテキストがどのように複数データに区切られているか（例: 改行 `\n` 区切り、カンマ `,` 区切りなど）をシートの実際のデータから判別、あるいは安全に分割（`split`）するロジックを実装してください。
2. 分割したデータを、`test-main.py` の `mock_data["news_list"]` と同じディクショナリ構造（`title`, `summary`, `script`, `image_path`）にパースし、既存の `generate_audio_timeline()` および `render_video()` に破綻なく流し込めるようにブリッジ（構造化）してください。
3. 現状、G列は「画像KW（またはファイル名）」となっているため、ローカルにある該当画像（`news1_pexels.jpg` など）のパスへ安全にマッピングするか、ファイルが存在しない場合のデフォルト画像へのフォールバックを処理してください。

### タスク 3: 処理完了後の YouTube URL 書き戻し（Update）

1. 動画のレンダリング、および YouTube への限定公開アップロード（`upload_to_youtube()`）が成功し、`video_id` が取得できたら、該当する動画URL（`https://youtu.be/{video_id}`）を生成する。
2. 処理対象となった行の **H列 (row[7])** に対して、生成した YouTube URL を `spreadsheets.values().update()` を用いて書き戻す処理を実装してください。
3. 書き戻しが成功したことを確認した上で、Discordへの通知処理へと進むこと。

## 5. 進め方のガイドライン

- **既存資産の尊重:** `test-main.py` に実装されている OAuth 2.0 の認証ロジック（`get_youtube_service` を拡張してマルチサービス化する等）や、Discordの通知、TTS生成ロジックは最大限流用し、データの入り口（インプット）と出口（URL書き戻し）をスプレッドシートへと繋ぎ込んでください。
- **レイアウトエラーの扱い:** MoviePy 2.0 の座標指定に関する内部エラー（`int has no len()`）の根本治療は、ユーザー指示により**意図的に後回し**としています。今回は「エラーは出るが処理が走り切る」現在の挙動を維持したまま、データソース連携の完成に集中してください。
- **ステップ・バイ・ステップでの検証:** まずスプレッドシートからターゲット行を正しく読み込んでパースできるかを出力（`print`）で確認し、問題なければ動画生成フェーズへ結合してください。

以上の要件を満たす `main.py` を作成（リファクタリング）してください。作業の開始をお願いします。
